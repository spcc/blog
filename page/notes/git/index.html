# git常用操作

# 创建ssh

ssh-keygen -t rsa -C "one@gmail.com"

这里得rsa是文件名

# 如何在远程某个分支的基础上新建分支

  1. git branch newBranch //新建本地分支
  2. git checkout newBranch //本地切换到自己的分支
  3. git pull origin xxxxBranch //从远程目标分支copy代码
  4. git push origin newBranch //将本地新建的分支同步到服务器
  5. git branch --set-upstream-to=origin/newbranch newbranch . //本地分支和远程分支建立追踪

# 如果远程已有分支，本地没有该分支

git checkout --track origin/branch_name

# git代码回滚

# 本地代码库回滚

git reset --hard commit-id :回滚到commit-id，讲commit-id之后提交的commit都去除 git reset --hard HEAD~3：将最近3次的提交回滚

# 远程代码回滚

这个是重点要说的内容，过程比本地回滚要复杂

应用场景：自动部署系统发布后发现问题，需要回滚到某一个commit，再重新发布

原理：先将本地分支退回到某个commit，删除远程分支，再重新push本地分支

操作步骤：

1、git checkout master

2、git pull

3、git branch master_backup //备份一下这个分支当前的情况

4、git reset --hard the_commit_id //把master本地回滚到the_commit_id

5、git push origin :master //删除远程 master

6、git push origin master //用回滚后的本地分支重新建立远程分支

7、git branch -d master_backup //如果前面都成功了，删除这个备份分支
