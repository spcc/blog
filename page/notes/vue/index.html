# 选项 / 资源

# filters

  • 参数：
    • {string} id
    • {Function} [definition]
  • 类型：Object
用法 
<!--使用过滤器-->
{{ dateStr | formatDate }}
filters: {
  formatDate(value) {
    return format(value, 'yyyy-MM-DD HH:mm:ss')
  }
}

// 注册全局过滤器
Vue.filter('my-filter', value => {
  // 返回处理后的值
  format(value, 'zh_CN')
})

// 建议注册全局过滤器步骤
// 对于全局过滤器，一般建议在项目里面添加filters目录，然后在filters目录里面添加
// filters\index.js
import Vue from 'vue'
import { format } from '@/utils/date'

Vue.filter('formatDate', value => {
  return format(value, 'yyyy-MM-DD HH:mm:ss')
})
// 然后将filters里面的文件引入到main.js里面

# vm.$set

  • 参数：
    • {Object | Array} target
    • {string | number} propertyName/index
    • {any} value
  • 用法
    this.$set( this.userInfo , 'officialAccount', '前端真好玩')
  • 使用场景
    改变数组索引。
Vue对数组的以下方法进行的包装变异:（否则不能监听到数组变动） 
// push
// pop
// shift
// unshift
// splice
// sort
// reverse

// 改变数组第一个数据 set
this.$set(this.list,0,'王五')

// splice
this.list.splice(0,1,'王五')

# 选项 / 其它

# model

  • 修饰符
    • .lazy change 事件_之后_进行同步 <input v-model.lazy="msg">
    • .number 将用户的输入值转为数值类型 <input v-model.number="age" type="number">
    • .trim 自动过滤用户输入的首尾空白字符 <input v-model.trim="msg">

# 实例方法 / 生命周期

# vm.$forceUpdate()

  • 示例
    迫使 Vue 实例重新渲染。注意它仅仅影响实例本身和插入插槽内容的子组件，而不是所有子组件。
    改了data之后，调用一下$forceUpdate就会让Vue组件重新渲染.这个方法并不建议使用，因为它会引起许多不必要的性能消耗
  • 用法
    this.$forceUpdate()
